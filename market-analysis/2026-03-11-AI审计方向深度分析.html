三大政策红利

  • 国资委审计集中化 (2024)：所有国企必须在集团层面集中审计机构、审计事项和审计人员，子公司不再单设审计部门。明确要求建设"业审一体"IT 平台

A

国企 / 上市公司内审平台

目标客户：央企、地方国企、上市公司内审部门 | 市场：338 亿+

客户画像

  • 组织架构：党委 → 审计工作领导小组 → 董事会审计委员会 → 总审计师（现为强制设置）→ 总部审计中心 → 区域分中心
  • 团队规模：员工总数的 0.2%（最少 5 人）。5000 人企业 = 10-20 人审计团队；50000+ 人央企 = 100-200+ 人
  • 现有工具：用友审友 A7（10 万+ 客户）、浪潮海岳（200+ 央企）、金蝶审计、Excel 仍为主要分析工具
  • IT 环境：SAP (24.7%) + 用友 NC/BIP (70% 央企) + 华为 MetaERP (35% H1 2024) + 金蝶 EAS，多数大型国企集团同时运行多套 ERP
  • 预算：小型国企审计系统 50-200 万；大型央企数字化审计平台 500-2000 万+；软件仅占项目成本 30-50%

核心痛点（按影响排序）

  • 覆盖不足：人工方法无法覆盖大型多层级集团的全部业务流程
  • 数据处理弱：海量数据分散在数十个系统，手工分析缓慢易错
  • 事后审计：风险在造成损失后才被发现，无法在"萌芽阶段"预警
  • 审计人才缺口：传统财务审计背景，缺乏数据分析和编程能力
  • 数据孤岛：审计与财务、采购、HR、法务之间缺乏统一数据共享机制
  • 集中化落地难：新规要求"上审下"，但如何高效远程审计分散子公司是新挑战

判断与建议切入策略

机会最大但门槛也最高。建议不做整体审计平台替代（打不过用友/浪潮），而是做AI 分析智能层——与现有审计平台集成，提供异常检测、智能底稿、持续监控等 AI 增强能力。

推荐路径：从能源/电力/电信等 AI 审计最先进的垂直行业切入（有参考客户），以 pilot 项目展示可量化指标（覆盖率提升、时间节约、风险检出率）。目标客单价 100-500 万。

金税四期——恐惧驱动的刚需

  • 138 个数据源联网：税务、银行、社保、海关等。为每个企业创建"数字孪生"
  • 自动风险检测：三流不一致（1 秒检出）、异常成本结构、高频大额私转（月 5 笔以上或单笔 50 万+）、增值税链断裂
  • 2026 执法转向"精准打击"：从"广撒网"变为 AI 定向稽查，重点行业：电商、高净值个人、跨境贸易
  • 90%+ 中小企业存在某种程度的合规风险，2024 年全国补缴税款 2.1 万亿

产品形态设想

层级功能频率
T1 实时监控费用异常检测（重复报销、超标）、发票合规（三流一致、增值税链）、税负率监控 vs 行业基准、现金流异常预警Always-on

数据接入路径

  • 金蝶 OpenAPI：完整 API 平台，凭证同步、财务报表、发票数据均可获取。SME 渗透率 30%+
  • 用友 Open API：开发者中心完善，支持 200+ 系统对接（钉钉/企微/SAP）
  • 柠檬云：API 能力有限，需要数据库备份文件做接口开发。封闭生态是障碍
  • 集成中间件：集简云（200+ 软件连接器）、腾讯轻联（金蝶官方连接器）、简道云可桥接

市场空白：没有一个产品同时覆盖"费用异常检测 + 税务风险预警 + 财务健康评分 + 自动报表审查"。税友聚焦税务合规，百望聚焦发票/交易，分贝通/易快报只做费控。

  • 免费层：基础税务风险提醒（获客工具）
  • 标准版 ~2000 元/年：全量合规监控（对标税友验证的价格点）
  • 高级版 ~2 万+/年：深度诊断 + 税优咨询 + 人工专家
  • 渠道：与代账公司合作（服务数百万微型企业的中间层）

C

会计师事务所外审工具

目标客户：10,665 家会计师事务所、40.5 万注册会计师

市场结构

  • 行业规模：总收入约 1212 亿元（2022），~10,665 家事务所，平均 1330 万/家
  • 头部集中：Top 100 占行业收入 55%+。安永（68.1 亿）、毕马威（51 亿）、立信（50 亿）、天健（37.3 亿）
  • 2024 年地震：普华永道因恒大事件被罚 4.41 亿 + 停业 6 个月，75% A 股客户流失（110 → 27），首次跌出 Top 100。1100+ 家 A 股公司更换审计师，创历史记录
  • 国内所崛起：立信、天健、容诚、致同吸收了大量普华客户，急需提升产能和工具效率

审计师最痛的点

  • 底稿编制占 40% 时间：手工数据录入、Excel 复制粘贴、跨表勾稽关系核对，大量重复劳动
  • 函证流转慢：纸质函证来回数周。电子函证平台已有 1095 家事务所 + 2019 家金融机构接入，但小银行仍纸质
  • 抽样局限：传统审计抽 25-60 笔样本；AI 可做全量检测——这是范式级升级
  • 年审季极端加班：12 月-4 月高强度，春节无休，"底稿做不完"是全行业共识
  • 人才流失严重：职业发展慢、强度大，年轻 CPA 大量离职

判断

痛点最强、时间窗口最好的方向。普华事件重塑市场格局，国内所急需工具提升产能。鼎信诺 AI 弱、四大工具不开放、DeepSeek 降低了部署成本。从"全量分录检测 + 智能底稿 + 报告起草"三件套切入，面向排名 10-200 的事务所（增长最快、最缺工具、预算中等）。

D

运营效率诊断（Claude + MCP）

创新方向：LLM 接入办公系统做全量运营扫描，定位"经营分析助手"

技术架构

LLM (Claude / DeepSeek) | +-------------------+-------------------+ | | | | | MCP MCP MCP MCP MCP 飞书/钉钉 项目管理 Git 文档系统 HR/OA (消息/日程 (Jira/ (Commit (飞书文档 (考勤/ 审批/日报) Tapd) /PR) 本地文件) 绩效) | | | | | +-------------------+-------------------+ | 全量运营数据画像 流程效率分析 异常模式检测 部门级诊断报告

定位策略（不叫"人效审计"）

核心原则：扫流程和部门，不扫个人；说"效率瓶颈"，不说"谁没干活"

不要说应该说原因
"张三三个月没产出""XX 部门产出效率低于行业均值 40%"避免个人对抗
"疑似贪污""该审批链平均耗时 5 天，行业标准 1 天"中层不抵触
"监控每个人""运营流程 X 光诊断"合规 + 接受度

可以发现的高价值异常

场景检测方式量化容易度敏感度
虚假报销 / 重复报销发票 OCR + 规则 + 异常检测高（金额明确）低
采购价格虚高市场价比对 + 历史价格趋势高中
审批流程冗余流程挖掘 + 时间分析高（时间可量化）低
会议效率低下日历分析 + 会后产出关联中低
关联交易 / 利益输送图网络分析供应商关系中高
人力空转 / 无效岗位工作量分析 + 产出对比低（容易扯皮）极高

商业模式——效果分润

  • 基础 SaaS 费：覆盖成本，按年收取
  • 效果奖金：发现并经客户确认的节省金额，分润 5-15%
  • "确认"标准必须预先定义：进入审计报告 / 财务确认 / 实际追回 —— 否则扯皮不断
  • 最佳切入场景：虚假报销 + 采购价格异常（金额清晰、证据链明确、客户容易认账）

关键风险与合规红线

《个人信息保护法》限制员工监控——扫描聊天记录、邮件、文档内容需明确告知并取得同意。建议：只扫描结构化数据（审批记录、提交时间、产出物元数据），不扫描内容本身。定位为"流程分析"而非"员工监控"。

买家是老板/CEO，但中层管理者会成为最大阻力（人效诊断首先暴露管理层问题）。需要从"帮管理层提效"而非"审查管理层"的角度包装。

共享底层能力

四条审计线共享底层 AI 能力，降低边际开发成本：

  • 异常检测引擎：分录异常、费用异常、交易异常——通用模型，按场景配置规则
  • 文档 NLP：OCR + 发票识别 + 合同关键条款提取——中小企业和事务所都需要
  • 报告生成：风险报告、审计底稿、合规评分——模板 + LLM 生成
